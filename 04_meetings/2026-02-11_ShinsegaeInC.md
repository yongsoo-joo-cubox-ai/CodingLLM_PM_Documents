## 1. 회의 개요
* **발표자(문기봉 상무):** AI를 활용하여 개발자의 의도(Intention)를 파악하고, 엔티티 모델을 기반으로 UI 및 비즈니스 로직 코드를 자동 생성하는 솔루션을 시연함.
* **참석자(신세계 I&C):** 현재 내부적으로 AI 개발 플랫폼(Sparos DevX 등)을 구축 중이며, 폐쇄망 환경에서의 개발 생산성 향상 방안을 모색 중인 개발/기획 그룹.

## 2. 주요 발표 및 시연 내용

### A. 솔루션 핵심 컨셉
* **개발 환경의 변화:** 기업 환경은 보안 문제로 클라우드 사용이 제한적이며, 비용 효율성을 위해 **경량화된 모델(Small Model)** 과 **온프레미스(On-premise)** 환경이 중요함.
* **의도(Intention) 중심 개발:** 단순 코딩이 아니라 개발자의 의도를 파악하는 데 AI를 사용하고, 나머지는 기존 소프트웨어 공학 기법을 결합하여 생산성과 품질을 확보함.
* **Spec-Driven (스펙 기반):** 채팅창에 프롬프트를 입력하는 방식(Prompt-Driven)보다, 정의된 스펙(ERD, 모델)을 기반으로 결과물을 만들어내는 방식이 기업 환경에 더 적합함.

### B. 시연 기능 (Demo)
* **코드 생성:**
    * 기존 ERD, SQL DDL, JSON Schema 등을 임포트하면 자동으로 모델이 생성됨.
    * 생성된 모델을 기반으로 UI 화면(리스트, 상세 조회 등)과 비즈니스 로직 코드를 즉시 생성.
    * X-Frame 5, Vue.js, React 등 다양한 프레임워크 지원 (현재 데모는 Vue.js 위주).
* **수정 및 커스터마이징:** 생성된 코드는 보일러 플레이트(Boilerplate) 수준이며, 개발자가 이후 비즈니스 로직이나 CSS 스타일을 수정할 수 있음.

## 3. 주요 질의응답 (Q&A) 및 기술 검토

### 💡 보안 및 인프라
* **Q (고객):** 금융권이나 기업 내부는 인터넷이 차단된 **폐쇄망**이다. 외부 API 호출 없이 내부에서만 동작 가능한가?
* **A (발표자):** 가능하다. 이 솔루션은 **vLLM(Virtual Large Language Model)** 을 사용하여 서버 3대(Web, App, Model) 구조로 구성되며, 모델을 내부에 설치하여 단독으로 운영할 수 있다.

### 💡 기존 시스템 호환성
* **Q (고객):** 우리는 넥사크로(Nexacro)나 웹스퀘어(WebSquare) 같은 국산 UI 툴을 많이 쓴다. 이것들과 호환되나?
* **A (발표자):** 해당 툴들은 경쟁 관계에 있으나, 기술적으로 XML 기반 등 표준을 따르므로 변환 로직을 구현하면 가능하다. 현재 뷰(View), 리액트(React) 등 오픈소스 기반은 바로 지원한다.

### 💡 커스터마이징 및 데이터 처리
* **Q (고객):** 글로벌 소싱을 하다 보면 컬럼명(First Name, Last Name 등)을 우리 식대로 바꿔야 한다. 가능한가?
* **A (발표자):** 가능하다. 표준 용어를 정의해두고 매핑하는 방식으로 처리하거나, 생성 후 수정할 수 있다.

### 💡 AI 모델 및 비용
* **Q (고객):** 모델은 어떤 것을 쓰는가? 비용 효율성은?
* **A (발표자):** 거대 언어 모델(LLM) 대신 특정 목적에 맞게 튜닝된 작은 모델을 사용한다. 할루시네이션(환각 현상)을 통제하기 쉽고, 기업 내부 서버에서도 무리 없이 돌아가도록 경량화했다.

## 4. 고객사 현황 및 피드백
* **고객사 상황:** 현재 'Sparos DevX'라는 자체 AI 개발 플랫폼을 만들고 있으며, 신세계 그룹 등 계열사에 적용하려 함.
* **고민점:** 폐쇄망 이슈로 인해 GitHub Copilot 같은 외부 도구를 쓰기 어렵고, 인터넷이 안 되는 환경에서 개발 생산성을 높여야 하는 과제가 있음.
* **피드백:**
    * 단순한 Chat 기반이 아니라, **프로젝트 단위의 스펙을 관리하고 코드를 생성해주는 접근 방식**이 인상적임.
    * 내부 개발자들이 반복 업무(Boilerplate code 작성)를 줄이는 데 도움이 될 것으로 보임.
    * 향후 VS Code 익스텐션 형태나 내부 플랫폼과의 연동 가능성을 열어두고 싶음.
