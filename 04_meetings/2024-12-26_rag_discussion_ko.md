## 1. 회의 배경 및 목표

금융/공공 분야를 대상으로 무상 AI 커버링 방안 검토. RAG 기반 코딩 어시스턴스와 코딩 자동화(Claude Code 등) 도입 가능성 논의.

---

## 2. 주요 논의 내용

### 2.1 시스템 아키텍처 구상

현재 구상 중인 시스템 구조:

- **프론트엔드**: xFrame + Dev Studio 기반 UI
- **백엔드**: GPT/LLM API 연동 + RAG + DB 구성
- **화면 정의**: Form XML 방식 활용

### 2.2 개발 범위별 비중 검토 (총 4개 영역)

| 구분 | 비중 | 내용 | 비고 |
|------|------|------|------|
| ① UI 생성 | 20% | Form XML 자동 생성, 파일명/위치/프로비 설계, xFrame 스펙 취합 | |
| ② API/정보 연동 | 20% | 정보이력 기반 기능 개발, 도움말 기능 | RAG 적용 대상 |
| ③ 코딩 어시스턴스 | 30% | 코드 작성 지원 | Cursor, Windsurf 참고 |
| ④ 테스트 케이스 | 20% | 테스트 케이스 자동 생성/적용 | |

### 2.3 기술 스택 검토

**UI 설계 방향**:

- Json → 파싱 (Java, HTML 등)
- 전자정부 프레임워크 + MVC 패턴 + 생성형 AI 연계
- XUL 기반 XML UI 설계 검토 ("SW base" 접근)
- xFrame GPT Chat형 인터페이스 확인 중

**AI 모델 검토**:

- Claude: 제약 있음 (비용 이슈)
- Gemini: 검토 중
- 기타 GPT 계열

### 3.1 AI 활용 관련 이슈

- **비용**: AI 박스(API) 비용 부담
- **출력 일관성 문제**: Data가 바뀌면 output이 달라짐 → 일정한 결과 보장 어려움

- **RAG + 러닝**: 현재 조합 적용은 어려움
