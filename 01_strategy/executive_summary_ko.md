## 0. 프로젝트 배경

### 프로젝트 비전

> **"국내 SI 사업을 위한 범용 AI 코딩 어시스턴스"** (like CursorAI / Claude Code for SI Project)

| 구분 | 내용 |
|------|------|
| **본질** | 범용적, 확장 가능한 SI용 AI 코딩 어시스턴스 |
| **첫 번째 응용 사례** | SOFTBase xFrame5 (공공 SI 프레임워크) |
| **확장 방향** | Spring, Vue, React, 전자정부 프레임워크 등 |

### 프로젝트 조직 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    프로젝트 총괄 / PO                         │
│                      문기봉 상무 (대보DX)                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────┐     ┌─────────────────────┐       │
│  │   PoC / 구현 파트    │     │   R&D / 상용화 파트   │       │
│  │   문기봉 상무 (대보DX)  │     │   시선AI             │       │
│  │                     │     │   서브PO: 주용수      │       │
│  └─────────────────────┘     └─────────────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

### 역할 분담

| 파트 | 담당 | 핵심 역할 | 산출물 |
|------|------|----------|--------|
| **PoC/구현** | 문기봉 상무 (대보DX) | 프로토타입 개발, 기능 구현, 데모 시연 | 동작하는 프로토타입, 고객 피드백 |
| **R&D/상용화** | 시선AI (서브PO: 주용수) | 모델 개발, LLM 배포, 인프라 구축, 솔루션화 | 상용 제품, 배포 인프라, 기술 문서 |

### 고객사 프로필

#### SOFTBase (소프트베이스) - 첫 번째 파일럿 고객

| 항목 | 내용 |
|------|------|
| **설립** | 2001년 6월 (금융업무개발 SI사업으로 시작) |
| **전환** | 2004년 UI솔루션 전문회사로 전환 |
| **대표제품** | **xFrame5** (HTML5 기반 UI개발 솔루션) |
| **시장포지션** | 제1금융시장 UI솔루션 선두주자 |
| **기술특징** | Any Device, Any Browser, Any Server Platform 지원 RIA 솔루션 |

**주요 레퍼런스**: KB국민은행, KEB하나은행, 우리은행, 새마을금고, 대법원, 철도공사, 국민연금관리공단, 삼성전자

## 1. 경영진 요약

> **핵심 인사이트**: Coder의 고유 가치는 "AI 코드 생성"이 아닌 **"AI 코드 거버넌스"**입니다. 생성된 코드가 기업 표준을 준수하도록 보장하는 것이 핵심 차별점입니다.

### 시장 포지셔닝

| 구분 | 소비자용 코딩 에이전트 | Coder |
|------|------------------------|-------|
| 대상 사용자 | 개인 창작 개발자 | 규제 기업 팀 개발자 |
| 환경 | 인터넷 접근, 자유로운 기술 스택 | 폐쇄망, 제한된 기술 스택 |
| AI 접근 | 클라우드 AI 서비스 | 온프레미스 LLM 전용 |
| 산출물 목표 | 개인 창의성 극대화 | 균일성 및 표준 준수 |

### Coder의 5대 USP (Unique Selling Points)

| # | USP | 설명 |
|---|-----|------|
| 1 | 결정론적 출력 품질 | 6단계 후처리 파이프라인으로 일관된 품질 보장 |
| 2 | 표준 강제 | 회사 규칙이 생성 시 자동 적용 (제안이 아닌 강제) |
| 3 | 완전한 온프레미스 | 외부 데이터 전송 제로 (설계 원칙) |
| 4 | 감사 추적 | 모든 생성 기록: 누가, 언제, 무엇을 |
| 5 | LLM 추상화 | 모델 변경 시 플러그인 코드 변경 없음 |

### 2.2 Cline Enterprise 분석 (중요!)

Cline은 **Enterprise 버전**을 통해 기업용 기능을 이미 제공하고 있습니다:

| 기능 | Cline Enterprise | Coder |
|------|------------------|-------|
| **SSO (Single Sign-On)** | ✅ 제공 | ✅ 제공 |
| **RBAC (역할 기반 접근 제어)** | ✅ 제공 | ✅ 제공 |
| **중앙 정책 관리** | ✅ 제공 | ✅ 제공 |
| **분석/감사 기능** | ✅ 제공 | ✅ 제공 |
| **결정론적 파이프라인** | ❌ 미제공 | ✅ **6단계 파이프라인** |
| **API 허용목록 강제** | ❌ 미제공 | ✅ **허용된 API만 생성** |
| **완전 온프레미스 설계** | △ 부분적 | ✅ **설계 원칙** |
| **국내 공공 SI 특화** | ❌ 미제공 | ✅ **xFrame5 최적화** |

> **전략적 시사점**: 단순한 "기업용 기능 부재"로 차별화할 수 없음. **결정론적 파이프라인**과 **완전 온프레미스**, **국내 SI 특화**가 진정한 차별점.

## 3. 현황 분석

### 강점

- LLM 추상화 우수 (9개 백엔드 지원)
- 6단계 후처리 파이프라인 (품질 보장)
- 온프레미스 LLM 검증 완료:
  - **GPT-OSS 20B**: 품질 94%, 응답 ~15초 (권장)
  - **Qwen2.5-Coder-32B-AWQ**: 품질 54%, 응답 25-65초 (동시 사용자에 따라 상이)
  - 상세: `03_development/2026-01-15_project_intro/model_benchmark.md` 참조
- 시맨틱 RAG 및 지식 베이스
- 금융권 보안 요구사항 충족
- **소프트베이스 xFrame5 파일럿 진행 중** (검증된 수요)

### 개선 필요 영역

- xFrame5에 강하게 결합된 생성 기능 → 프레임워크 분리 필요
- 파일 편집, 터미널 실행 등 기본 기능 부재
- Eclipse 전용 (VS Code 미지원)
- CLI 인터페이스 부재

## 6. 전략적 권고사항

> **포지셔닝 전략**:
> "규제 산업의 기업 개발팀을 위해, 코드 품질과 컴플라이언스를 보장해야 하는 환경에서, **Coder는 표준을 강제하고 완전한 감사 추적을 제공하는 AI 코드 거버넌스 플랫폼**입니다. 개발자 자유를 극대화하는 창작 코딩 도구와 달리, Coder는 모든 생성 산출물이 기업 요구사항을 충족하도록 보장합니다."

### 추진해야 할 것

- 파일 작업 + 검증 게이트웨이 (테이블 스테이크)
- CLI 인터페이스 (배치, CI/CD, 자동화)
- 프레임워크 분리 (시장 확장)
- 멀티 모델 라우팅 (유연성, 안정성)
- **Cline Enterprise 대비 차별화 강화** (결정론적 파이프라인 홍보)

### 피해야 할 것

- MCP 확장성 (통제되지 않는 도구 접근)
- 브라우저 자동화 (범위 외)
- 페르소나/테마 (소비자 기능)
- 완전 자율 터미널 (거버넌스 훼손)
