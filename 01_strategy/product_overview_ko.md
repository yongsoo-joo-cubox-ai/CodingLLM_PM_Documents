## 개요

Coco(Coordinated Coding)는 **AI 코드 거버넌스 플랫폼**으로, 폐쇄망 환경에서 운영되는 온프레미스 AI 코드 생성 솔루션입니다. 단순한 AI 코드 생성이 아닌, 기업 표준 준수와 감사 추적이 가능한 **거버닝된 코드 생성**을 제공합니다.

### 제품 정체성

| 구분 | 설명 |
|------|------|
| **이름** | Coco (Coordinated Coding) |
| **정체성** | AI 코드 거버넌스 플랫폼 |
| **대상** | 폐쇄망, 온프레미스 LLM, 규제 산업 기업 (금융, 공공, 대기업 SI) |
| **핵심 가치** | 기업 코딩 표준 자동 적용 + AI 생성 코드의 완전한 추적 가능성 |

## 2. 핵심 기능

### 2.1 코드 생성 (Code Generation)

자연어 프롬프트 또는 스펙 기반으로 프레임워크별 코드를 자동 생성합니다.

**두 가지 생성 전략**:

| 전략 | 파이프라인 | 특징 | 권장 |
|------|-----------|------|------|
| **CGF-A** (Direct) | LLM → 직접 코드 생성 → Post-processing | 유연하지만 LLM 의존도 높음 | 단순 작업 |
| **CGF-B** (Spec-first) | LLM → 스펙(UASL/SUIS) 생성 → MCP 서버가 코드 생성 | 65% 속도 개선, 3:1 품질 우위 | ✅ **권장** |

**6단계 결정론적 파이프라인**:

```
1. 입력 정규화 → 2. 컨텍스트 주입 → 3. LLM 생성
        → 4. 구문 검증 → 5. 표준 강제 → 6. 감사 로깅
```

**지원 프레임워크**:

| 프레임워크 | 코드 생성 | 프리뷰 | MCP 서버 |
|-----------|----------|--------|----------|
| **xFrame5** | ✅ XML + JS | ✅ 런타임 프리뷰 | xframe5-compiler |
| **Vue3** | ✅ SFC (.vue) | ✅ 런타임 프리뷰 | vue-compiler |
| **Spring** | 📋 Phase 2 | - | 📋 계획 중 |
| **React** | 📋 Phase 3 | - | 📋 계획 중 |

### 2.2 코드 리뷰 (Code Review)

생성된 코드 또는 기존 코드에 대해 AI 기반 품질 분석을 수행합니다.

| 리뷰 항목 | 설명 |
|-----------|------|
| **구문 검증** (Syntax) | 프레임워크별 올바른 구문 사용 여부 |
| **패턴 준수** (Patterns) | 기업 코딩 표준 준수 여부 |
| **명명 규칙** (Naming) | ID, 변수명 등 명명 규칙 준수 |
| **보안** (Security) | 보안 취약점 탐지 |
| **성능** (Performance) | 성능 이슈 탐지 |

### 2.3 Q&A (프레임워크 지식 질의응답)

프레임워크 지식 베이스를 기반으로 개발자 질문에 정확한 답변을 제공합니다.

| 특징 | 설명 |
|------|------|
| **RAG 기반** | 지식 베이스 의미 검색 (all-MiniLM-L6-v2) |
| **환각 방지** | "추론 말고 추출" 원칙 — 지식 베이스에 없는 내용은 답변하지 않음 |
| **복수 후보** | 동일 스코어 답변이 여러 개일 때 사용자에게 선택 제공 |
| **코드 예시** | 답변에 실행 가능한 코드 예시 포함 |

### 2.4 코드 프리뷰 (Code Preview)

생성된 코드를 즉시 실행하여 결과를 미리 볼 수 있습니다.

| 기능 | 설명 |
|------|------|
| **실시간 프리뷰** | 생성 코드를 별도 창에서 실행 |
| **Mockup 데이터** | 백엔드 없이 가짜 데이터로 UI 확인 |
| **인터랙션 테스트** | 검색 버튼 등 UI 인터랙션 동작 확인 |

## 3. UASL/SUIS — 프레임워크 중립 UI 언어

### 3.1 개요

UASL(Universal Application Specification Language)과 SUIS(Standardized UI Specification)는 프레임워크에 종속되지 않는 UI 기술 언어입니다.

```
사용자 의도 → LLM → UASL/SUIS 스펙 → MCP 서버 → 프레임워크별 코드
                                         ↓
                                  xFrame5, Vue3, Spring, React ...
```

### 3.2 장점

| 장점 | 설명 |
|------|------|
| **프레임워크 독립** | 하나의 스펙으로 다양한 프레임워크 코드 생성 |
| **LLM 부하 감소** | LLM은 스펙만 생성 → 코드 변환은 프로그램이 처리 |
| **품질 보장** | 코드 생성이 프로그램 기반이므로 결정론적 품질 |
| **확장 용이** | 새 프레임워크 추가 시 MCP 서버만 추가 |

## 4. 6대 USP (Unique Selling Proposition)

| # | USP | 설명 |
|---|-----|------|
| 1 | **결정론적 출력** | 동일 입력 → 동일 출력 보장. 6단계 파이프라인으로 랜덤성 제거 |
| 2 | **표준 강제** | 기업 코딩 표준 자동 적용. 표준 위반 코드가 출력되지 않음 |
| 3 | **완전한 온프레미스** | 외부 데이터 전송 제로. 폐쇄망에서 단독 운영 가능 |
| 4 | **Spec-Driven 생성** | LLM은 스펙만 생성, 코드는 MCP 서버가 프로그램으로 생성 → 투명하고 예측 가능 |
| 5 | **감사 추적** | 모든 생성 과정 로깅. 규제 컴플라이언스 충족 |
| 6 | **LLM 추상화** | 특정 LLM에 종속되지 않음. 모델 교체 시 코드 변경 불필요 |

## 6. 접근 방법

### 6.1 Coco Studio (웹)

- **URL**: https://coco.secernai.net (시선AI 내부망)
- **기능**: 코드 생성, 코드 리뷰, Q&A, 코드 프리뷰
- **대상**: 일반 개발자, 기획자

### 6.2 Coco CLI (터미널)

```bash
# 코드 생성
coco generate --product xframe5-ui --prompt "회원 목록 화면"

# 배치 생성
coco batch --product xframe5-ui --input specs.json

# 모델 목록
coco models
```

### 6.3 Eclipse Plugin

- Eclipse IDE에서 직접 코드 생성/리뷰 사용
- SI 프로젝트에서 가장 많이 사용하는 IDE 지원

### 6.4 API

- RESTful API로 직접 통합 가능
- 상세: [API 레퍼런스](../02_implementation/api_reference_ko.md)

## 7. 타겟 고객 및 사용 시나리오

### 7.1 타겟 고객

| 세그먼트 | 대표 고객 | 핵심 니즈 |
|----------|----------|----------|
| **금융권** | 은행, 증권, 보험 | 망분리 환경에서 AI 코딩 도구, 감사 추적 |
| **공공기관** | 정부, 공기업 | 국산 SW, 온프레미스, 보안 인증 |
| **대기업 SI** | 신세계 I&C, 삼성SDS 등 | 반복 Boilerplate 감소, 프로젝트 표준화 |

### 7.2 사용 시나리오

**시나리오 1: 금융 SI 프로젝트**

> 은행 차세대 시스템 개발 시, xFrame5 기반 수백 개 화면을 생성해야 하는 경우

1. 엔티티 모델(ERD) 임포트
2. Coco가 UASL/SUIS 스펙으로 변환
3. MCP 서버가 xFrame5 XML + JS 코드 생성
4. 프리뷰로 즉시 확인
5. 감사 로그에 생성 이력 기록

**시나리오 2: 대기업 SI 내부 플랫폼**

> 신세계 I&C처럼 자체 개발 플랫폼에 AI 코드 생성 기능을 통합하는 경우

1. Coco API를 내부 플랫폼(Sparos DevX 등)에 연동
2. 프로젝트 설정에 따라 Vue3 또는 xFrame5 코드 자동 생성
3. 개발자는 Boilerplate 대신 비즈니스 로직에 집중
