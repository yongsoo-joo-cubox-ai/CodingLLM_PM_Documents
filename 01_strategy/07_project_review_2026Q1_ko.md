## 1. 프로젝트 개요

### 1.1 비전

> **"국내 SI 사업을 위한 온프레미스 AI 코드 거버넌스 & 코딩 에이전트 플랫폼"**

### 1.2 대외 브랜딩

| 항목 | 내용 |
|------|------|
| **대외 브랜드** | IntraGenX — "AI 기반 차세대 시스템 개발 플랫폼" |
| **The Brain** | 시선AI — LLM 개발, 모델 최적화, 인프라 |
| **The Body** | 대보DX — 애플리케이션, 어플라이언스, PoC 구현 |
| **확정 시점** | 2026년 3월 |

### 1.3 투트랙(Two-Track) 전략

| 구분 | 트랙 1: IntraGenX (Coco) | 트랙 2: SecernCode |
|------|--------------------------|-------------------|
| **접근법** | Top-Down (설계→스펙→코드 일괄 생성) | Bottom-Up (개발자 주도 자율형 에이전트) |
| **대상 사용자** | SI 프로젝트 PM·아키텍트 | 개발자 개인 |
| **제품 형태** | Coco Studio(웹), Coco CLI, Eclipse 플러그인 | Go 단일 바이너리(~50MB), TUI/WebUI/CLI |
| **핵심 기술** | UASL/SUIS + MCP 서버 + CGF-B 파이프라인 | Go 1.24 + 에이전틱 루프 + MCP |
| **주도 조직** | 대보DX + 시선AI | 시선AI |

### 1.4 조직 구조

```
프로젝트 총괄 (PO): 문기봉 상무 (대보DX)

    │   └ Phase 1 전체 구현, 데모 시연

    └─ R&D/상용화 파트: 시선AI
        └ 서브PO: 주용수
        └ Phase 1: 서포트 (0.6 FTE)
        └ Phase 2~: 본격 개발 (3.6 FTE)
```

> **영향**: 두 트랙이 서로 다른 모델 세대를 사용 중. vLLM 인프라에서 두 모델을 동시 서빙해야 하므로 GPU 메모리 및 운영 복잡도에 영향. Infra-S1(LiteLLM 멀티 모델)에서 해결 예정이나, 통합 시점이 미확정.

### 5.4 핵심 리스크

- **Phase 6 라이선스 보호 구현 범위**: License Agent + 3계층 검증은 Infra-S0, S1, S2에 걸치는 범위. 명확한 Stage 간 경계 필요
- **금융권 보안 심사**: Tensorizer + libsodium 암호화가 금융권 보안 심사 수준 충족 여부 미확인 (국정원 인증 필요 시 2~4주 추가)
